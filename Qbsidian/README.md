# 《程序设计基础》大作业功能设计文档

## 一、 项目名称
**Qbsidian：基于 Qt 的知识图谱与 Markdown 笔记软件**

## 二、 项目背景与立意

传统的笔记软件多采用“树状目录”进行线性存储，本项目旨在利用 C++ 与 Qt 框架进行拓展，开发一款轻量级的本地个人知识管理工具。借鉴 Obsidian 的核心理念，本项目将重点实现**双向链接**与**可视化知识图谱**，将离散的 Markdown 笔记交织成知识网络。同时对记录的笔记内容提供知识库检索、复习规划、例题抽查练习等功能。

本项目由小组成员们在程设复习中受到启发后设计，预计成果具有一定的生活与学习价值，且对课程中**面向对象设计、文件 I/O，以及数据结构、字符串处理和GUI 设计**的内容均能有所提升。

## 三、 核心功能设计

本项目功能模块划分为以下四大板块：

### 1. 基础工作区：双栏 Markdown 编辑与文件管理
*   **本地知识库：** 通过树状视图展示用户指定的本地文件夹结构，支持多层级目录的浏览与点击切换。
*   **双栏实时渲染引擎：** 界面采用左右分栏设计。左侧为纯文本编辑区，右侧为 Markdown 渲染预览区。用户在左侧输入时，右侧实时解析并渲染基础的 Markdown 语法（加粗、标题、列表等）。
*   **持久化存储：** 提供新建、保存、另存为功能，所有笔记以纯 `.md` 文件格式保存在本地，不依赖专有数据库。

### 2. 双向链接机制

*   **链接语法识别：** 程序实时监控文本输入，利用正则表达式捕获 `[[笔记名称]]` 格式的特定语法。
*   **自动跳转逻辑：** 在右侧预览区，被 `[[ ]]` 包裹的文本将高亮显示为超链接。用户点击该链接时，程序将拦截点击事件，自动在本地目录中检索对应文件，并将其加载到当前编辑区。若文件不存在，则提示用户一键创建。

### 3. 知识库检索

*   **全局检索**： 提供全局搜索侧边栏，支持对知识库目录下所有文本文件和附加`#标签`进行关键字遍历匹配。
*   **结果可视化展示：** 在检索结果栏中展现包含目标标签的文件名及包含关键字的上下文摘要，点击摘要即可快速定位至对应文件的具体行。

### 4. 基于力导向算法的动态知识图谱

*   **图结构数据提取：** 后台静默遍历所有笔记，提取双向链接关系，在内存中构建以笔记为“节点”，链接关系为“边”的图数据结构。
*   **物理引擎动态渲染：** 引入**力导向图算法**，模拟物理学中的排斥力与弹簧引力。打开图谱界面时，节点会经历动态的互相排斥与拉扯，最终呈现出结构清晰、无重叠的网状星空图，直观展示用户知识脉络的聚集度。

### 5.复习规划器

+ **复习进度追踪：** 系统在后台为每篇笔记维护一个学习状态记录，记录笔记的“首次创建时间”与“最近复习时间”。
+ **智能提醒与规划：** 引入间隔重复理念（可参考艾宾浩斯遗忘曲线），程序启动时自动计算当天需复习的笔记列表（如：1天、3天、7天、15天前的笔记），在主页形成“今日待办”复习流，提醒用户及时回顾。
+ **自定义复习计划**：用户可以选择不同复习模式，如“指定日期提醒（考前3天）“、“固定周期循环（每周末复习一次）”，或者完全自己安排复习日程表。

### 6.题目抽查练习系统

+ **自定义题库提取：** 用户在做笔记时，可使用自定义语法记录知识点。程序利用正则表达式后台扫描，自动将这些语句抽取为“复习题库”。
+ **抽查测试：** 结合复习进度规划，进入“复习模式”时，系统弹出独立测试窗口。正面显示问题，用户点击“查看答案”后核对，并反馈“认识/不认识”。答题情况影响该知识点在下一轮规划中的权重。



## 四、 开发计划

1.  **第一阶段：** 完成 UI 基础框架搭建，实现本地文件目录树，完成左侧纯文本输入、右侧 Markdown 实时同步渲染的双栏编辑功能，确保系统具备最基础的笔记功能。
2.  **第二阶段：** 实现 `[[ ]] `双链语法与 `#标签 `的自动识别，完成通过超链接点击进行文件间的无缝跳转功能；完成跨文件的全局关键字搜索与标签分类检索功能。
3.  **第三阶段：** 集中开发复习功能。利用正则提取笔记内的特定问答语法生成习题库；实现复习规划功能。
4.  **第四阶段：**提取双链数据构建动态知识图谱；测试用例并修复 Bug；美化 UI。
